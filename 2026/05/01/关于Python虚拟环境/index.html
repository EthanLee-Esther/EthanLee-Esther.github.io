关于Python虚拟环境

1. 什么是虚拟环境？原理？

1.1 什么是虚拟环境？

虚拟环境（Virtual Environment） 是 Python 提供的一种机制，它允许你在一台机器上创建多个相互隔离的 Python 运行环境。每个虚拟环境都有自己独立的：

  • Python 解释器（可以是不同版本）
  • 第三方包（pip 安装的库）
  • 脚本和可执行文件

💡 一句话理解：虚拟环境就像是一个”沙盒”，每个项目都可以在自己的沙盒里安装所需的依赖包，互不干扰。

1.2 为什么需要虚拟环境？

场景（不用虚拟环境） 问题
项目 A 需要 requests==2.25.0，项目 B 需要 requests==2.30.0 版本冲突
直接在系统 Python 中安装包 可能污染系统环境，甚至破坏系统工具
多人协作时依赖不一致 “在我电脑上能跑啊！”
不同项目需要不同 Python 版本 无法同时满足

1.3 虚拟环境的原理

虚拟环境的本质是**”目录隔离”**，而非真正的”虚拟化”。当你创建一个虚拟环境时，Python 会做以下事情：

核心原理：

  1. PATH 劫持：激活虚拟环境后，系统会临时修改 PATH 环境变量，把 venv/bin（或 Scripts）放到最前面，这样运行 python 时实际调用的是虚拟环境中的解释器。
  2. site-packages 隔离：虚拟环境的 site-packages 优先于全局 site-packages。虚拟环境中的 pyvenv.cfg 文件中有一行 include-system-site-packages = false，意味着不会继承全局包。
  3. 轻量级”拷贝”：虚拟环境不会完整复制 Python 解释器，而是创建一个指向系统 Python 的符号链接（或拷贝可执行文件），所以体积很小（通常几 MB）。
