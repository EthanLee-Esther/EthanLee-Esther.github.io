一、Docker 是什么？为什么用它？

一句话： 把应用和它的运行环境打包成一个轻量级、可移植的容器，走到哪跑到哪。

跟虚拟机的区别：

对比项 虚拟机 Docker 容器
内核 每个VM一个完整内核 所有容器共享宿主机内核
启动速度 分钟级 秒级，甚至毫秒级
资源占用 GB 级 MB 级
隔离性 硬件级隔离 进程级隔离（够用）
镜像大小 通常 GB 通常 MB

核心概念：

  • 镜像（Image）：只读模板，相当于”安装包”。比如 nginx:latest 就是一个镜像。
  • 容器（Container）：镜像跑起来的实例，相当于”运行中的程序”。一个镜像可以跑多个容器。
  • 仓库（Registry）：存镜像的地方。Docker Hub 是官方的，你也可以搭私有的。
  • Dockerfile：构建镜像的”配方”，一行一行写怎么打包。
  • Docker Compose：编排多个容器的工具，一个 YAML 搞定整个应用栈。

7.6 最佳实践

  1. 用 Alpine 或 slim 镜像：python:3.11-slim 比 python:3.11 小好几倍
  2. 合并 RUN 指令：减少镜像层数，RUN a && b && c 而不是三个 RUN
  3. COPY 之前先 COPY package.json：利用缓存，依赖没变就不重新安装
  4. 不要用 latest 标签：锁定版本，否则哪天构建出来就不一样了
  5. 不要在镜像里存敏感信息：用环境变量或 secrets 注入

十三、安全建议

  1. 不要用 root 跑应用：Dockerfile 里加 USER app
  2. 不要把 Docker socket 挂进容器：等于给容器 root 权限
  3. 镜像用固定版本：别用 latest
  4. 定期更新基础镜像：修复安全漏洞
  5. 用 .dockerignore：别把 .git、密钥文件打包进去
  6. 扫描镜像漏洞：docker scout cves my-image
