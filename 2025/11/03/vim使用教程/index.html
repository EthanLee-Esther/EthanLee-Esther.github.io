全宇宙最强编辑器——Vim

一、Vim 简介

1.1 什么是 Vim

Vim 是一款功能强大、高度可定制的文本编辑器，其名称 “Vim” 是 “Vi IMproved” 的缩写，意为 Vi 编辑器的改进版。它最初由 Bram Moolenaar 于 1991 年开发，作为 Unix 系统中经典编辑器 Vi 的增强版本。Vim 以高效文本操作和高度可定制性著称，被广泛应用于程序员、系统管理员等技术领域，甚至被称为“编辑器之神”。

1.2 Vim 的核心特点与优势

1.2.1 模式化操作

Vim 最显著的特点是区分不同操作模式，每个模式对应特定功能，避免键盘输入歧义。

  • Normal 模式（普通模式）：默认模式，用于浏览文本、执行命令（如复制、删除、查找）。
  • Insert 模式（插入模式）：用于输入文本，类似普通编辑器的编辑状态。
  • Visual 模式（可视化模式）：用于选中字符、行或块，配合命令批量操作。
  • Command 模式（命令模式）：输入 :command 执行全局操作（如保存、退出、查找替换）。

1.2.2 键盘快捷键体系

几乎所有操作均可通过键盘完成，减少鼠标依赖，提升操作速度。例如：

  • dd 删除当前行
  • p 粘贴剪贴板内容
  • /keyword 搜索关键词

1.2.3 强大的文本处理能力

  • 支持正则表达式查找替换（如 :s/old/new/g）。
  • 批量编辑（如通过 Ctrl+v 进入块选择模式，批量添加前缀）。
  • 多文件编辑与会话管理（:tabnew 新建标签页，:session 保存编辑状态）。

1.2.4 高度可定制性

  • 通过 .vimrc 配置文件自定义快捷键、主题、插件等（例：map <F2> :w<CR> 将 F2 绑定保存）。
  • 支持丰富的插件生态（如通过 Vundle、Pathogen 管理插件）。

1.2.5 轻量与高效

无需图形界面，可在终端中运行，资源占用极低，适合服务器环境。

1.3 Vim 与其他编辑器的对比

维度 Vim VS Code/Sublime Text Notepad++
学习曲线 陡峭（需记忆快捷键） 平缓（图形化操作） 极平缓（面向新手）
效率上限 极高（熟练后） 高（依赖插件） 中（适合简单文本）
使用场景 代码开发、系统管理 现代开发、图形化需求 Windows 日常文本编辑
扩展性 插件生态成熟（如 YouCompleteMe 补全） 插件丰富（可视化配置） 有限
跨平台支持 全平台（Linux/macOS/Windows） 全平台 Windows 专属

三、Vim 的基本操作与功能

3.1 Vim 的核心模式

3.1.1 命令模式（默认进入）

启动 Vim 就处在这，能执行光标移动、内容增删等基础操作。

常用操作示例：

  • 光标移动：h（左）、j（下）、k（上）、l（右），像玩游戏控制方向键；
  • 内容操作：yy 复制当前行，dd 删除当前行 ，比如写代码删冗余行超方便。

3.1.2 插入模式（命令模式按 i/a/o 进入）

想直接输入文字就切这，和普通编辑器 打字一样。

  • i ：光标前插入内容；
  • a ：光标后插入内容；
  • o ：光标下方新开一行插入 。

比如写代码注释，切插入模式直接敲文字。

3.1.3 底行模式（命令模式按 Shift + : 进入）

执行保存、退出、替换等“指令式”操作。

常用命令：

  • 保存退出：:w 保存，:wq 保存并退出 ；
  • 全局替换：: %s/old/new/g ，把文档里所有 old 换成 new ，改配置文件批量替换超好用；
  • 分屏操作：:sp 文件名 水平分屏，:vsp 文件名 垂直分屏 ，多文件对比编辑很方便，按 Ctrl + w 配合方向键切换分屏 。

3.1.4 替换模式（命令模式按 Shift + R 进入）

进入后直接输入会替换光标所在位置字符，按 Esc 回命令模式，适合快速替换少量字符场景。

3.2 光标移动

3.2.1 单字符移动

  • h：向左移动一个字符。
  • l：向右移动一个字符。
  • j：向下移动一行。
  • k：向上移动一行。

3.2.2 括号匹配跳转

  • %：在匹配的括号间跳转（适用于 ()、[]、{} 等结构）。

3.2.3 屏幕滚动

  • Ctrl + d：向下滚动半屏。
  • Ctrl + u：向上滚动半屏。
  • Ctrl + f：向下滚动一屏。
  • Ctrl + b：向上滚动一屏。
  • :n：跳到第 n 行（如输入 :10 跳到第 10 行）。
  • gg：跳到文件的开头。
  • G：跳到文件的结尾。

3.2.4 行内移动

  • 0：移动到当前行的行首。
  • ^：移动到行首第一个非空字符。
  • $：移动到行尾。

3.2.5 单词移动

  • w：跳到下一个单词的开头。
  • e：跳到当前单词的结尾，或下一个单词的结尾。
  • b：跳到前一个单词的开头。

3.3 文本删除

  • x ：删除当前字符。
  • dd ：删除当前行。
  • dw ：删除从光标位置到单词末尾。
  • d$ ：删除从光标位置到行尾。
  • d0 ：删除从光标位置到行首。
  • dgg ：删除从光标到文件开头。
  • dG ：删除从光标到文件结尾。

3.4 文本复制与粘贴

  • yy ：复制当前行。
  • y ：复制选定区域（可与可视模式结合）。
  • yw ：复制从光标到单词末尾。
  • y$ ：复制从光标到行尾。
  • p ：粘贴到光标后。
  • P ：粘贴到光标前。

3.5 撤销与重做

  • u ：撤销上一步操作。
  • Ctrl + r ：重做撤销操作。

3.6 替换

  • r+单个字母 ：用 单个字母 替换当前字符。
  • R ：进入替换模式，覆盖原文字。

3.7.1 文件操作

  • :w 保存文件。
  • :q 退出文件。
  • :wq 保存并退出。
  • :q! 强制退出。
  • :sav new_filename 另存为。

3.7.2 搜索与替换

3.7.2.1 搜索
  • /text ：向下搜索 text 。
  • ?text ：向上搜索 text 。
  • n ：跳到下一个匹配项。
  • N ：跳到上一个匹配项。
3.7.2.2 替换
  • :s/old/new/ ：将当前行的第一个 old 替换为 new 。
  • :s/old/new/g ：将当前行的所有 old 替换为 new 。
  • :%s/old/new/g ：将整个文件中的 old 替换为 new 。
  • :%s/old/new/gc ：替换前逐一确认。

3.8 多文件编辑与会话管理

  • :tabnew 新建标签页。
  • :session 保存编辑状态。

3.9 高级编辑技巧

3.9.1 精准打开定位

vim 文件名 +n ，打开文件直接跳到第 n 行 ，比如看日志 vim log.txt +100 ，直接定位到第 100 行报错位置。

3.9.2 调用系统命令

底行模式用 :!外部命令 ，比如 :!ls 查看当前目录文件，编辑时不用切终端。

3.9.3 批量编辑

通过 Ctrl+v 进入块选择模式，批量添加前缀。

四、Vim 插件推荐与使用

4.1 插件管理器

4.1.1 Vundle.vim

最常用最老牌最简单的 Vim 插件管理器，缺陷是没法并发执行更新、安装。

4.1.2 dein.vim

日本人 Shougo 写的，也是 neobundle.vim 的插件管理器的作者（neobundle 基本不维护了） ，此插件支持异步。

4.1.3 vim-plug

韩国人 junegunn (Junegunn Choi) 写的， 目前最快插件管理器，无论是更新，安装，全都是并发的。

4.2 文件，代码搜索工具

4.2.1 fzf

同样是韩国人 junegunn (Junegunn Choi) 写的，目前最快的文件搜索器，使用 Go 语言实现，几万个文件秒搜。

4.2.2 ag.vim

从 ack.vim 上 clone 出来的分支，搜索引擎使用了 the_silver_searcher，可以快速批量搜索代码，搜索文件，模糊匹配，正则表达式。

4.2.3 ctrlp.vim

原作者 kien 已经四年不维护这个插件了，目前是基友们在维护着，ctrlp 可以说是最老牌的 vim fuzzy 搜索的插件，用 ctrlp 自带功能 进行搜索其实速度很慢，但是它一旦搭配上如上的 ag.vim， 感觉就完全不一样了。

4.3 自动补全

4.3.1 neocomplete

日本人 Shougo 写的，比较老牌的自动补全插件，综合体验一般般，需要 Vim 支持 Lua。

4.3.2 YouCompleteMe

前 Google 的 C++ 工程师 Valloric (Val Markovic) 开发的一款，专门用于 C 系（C,C++,C#) 的自动补全工具，优点是速度非常快，而且支持变量跳转，函数关联跳转，检测代码报错，可以说是最牛逼的补全工具，缺点是太难配置，对于除了 C 系语言的其他语言补全支持并不好。对于刚接触 Vim 的同学来说相当不友好，不过文档比较齐全。

4.4 代码检测，代码对齐，格式化

4.4.1 ale

只支持 vim8.0，因为检测动作是异步完成的，所以基本（绝对）不会卡，8.0 以下的 vim 版本大家可以考虑升级一下自己的 vim 了。

4.4.2 tabular

老牌的代码对齐插件，非常好用，需要美化代码，排版等，这个插件绝对是神器。可以将常用的格式，排版方式做成键位映射，更加方便，速度感人。

4.4.3 vim-easy-align

同样是韩国人 junegunn (Junegunn Choi) 写的， 代码排版，格式化插件， 它参考了 tabular ，并且集成了 tabular 的优点，同时集成了自己的一套规则方法，有一定的学习成本。

4.5 装逼必备

4.5.1 vim-airline

各种 line 已经成为过去，基本 airline 已经实现了统一 ，同时支持各种插件，当然带来的负担就是会导致 vim 启动慢了一点。当然如果要用 vim 装逼的话，这个是必不可少的。

4.5.2 limelight.vim

同样是韩国人 junegunn (Junegunn Choi) 写的，可以实现关灯看小说的效果。

4.6 极速跳转

4.6.1 vim-sneak

我们知道，vim 中 F 和 f 键位只能快速定位当前行，但是 sneak 可以使其完全支持多行 f 和 F 进行跳转。

4.6.2 incsearch.vim

作者 haya14busa (haya14busa) 是个日本萌妹， incsearch 是增强 vim 中自带的 ? 和 ／ 搜索功能， 并且支持更加高级的正则表达式匹配，vim 默认搜索是只能高亮一个当前匹配的字符，但是 incsearch 却可以同时高亮所有匹配的字符。

4.6.3 vim-easymotion

老牌的文本快速跳转插件，极速定位光标，不过快捷键太不人性化。

4.7 版本控制 ( Git , Svn )

4.7.1 vim-signify

同时支持 Git 和 Svn ，速度也是相当不错的，高亮当前修改。比较全面的一个插件。

4.7.2 vim-gitgutter

只支持 Git 的高亮，速度也非常不错。

4.7.3 gitv

展开项目的 Git Tree 相当屌的一个插件，当然需要 vim-git 和 vim-fugitive 的支持。

4.8 改善生活

4.8.1 tpope (Tim Pope) 系列

  • vim-pathogen 插件管理器，geek 风格的。
  • vim-surround 快速插入， 绝对是个改善生活的插件！，给快速插入 [ ] {} “” ‘’ 将文本包裹起来。
  • vim-dispatch 你想象一下，在 Vim8.0 之前的版本还不支持异步的情况下，这个东西是怎么完成后台异步执行任务的？
  • vim-commentary 快速注释插件，绝对改善生活，但是在 HTML 文件中还是有小 bug 的。

4.8.2 terryma (Terry Ma) 系列

  • vim-multiple-cursors 见过 sublime 的多光标输入吧，那么这下 vim 也有，当然 vim 原生也支持多光标，不过这插件简化了操作。
  • vim-expand-region 快速选择文本， 每次使用 V+hjkl 来选择文本是不是很累？
  • vim-smooth-scroll vim 中的平滑滚动。

4.9 其他实用插件

4.9.1 coc.nvim

这是一个改变了 Vim/Neovim 生态的重要插件。它基于 Node.js 强大的执行能力，不仅实现了类似 VS Code 级别的智能补全和语言支持，还能直接复用 VS Code 的扩展生态。其功能覆盖广泛，包括代码补全、语法检查、模糊查找、文档检索等，几乎是一个全能型插件。

4.9.2 LeaderF

它是一款出色的模糊查找工具。体积小巧但功能强大，响应速度快，界面也十分简洁美观。值得一提的是，它对 Vim 7.4 版本也有良好的支持，尤其在 Windows 平台下表现出色。

4.9.3 indentline

用来显示缩进线的工具，帮助开发者更好地理解代码结构，尤其在处理多层缩进时非常实用。

4.9.4 vim-translator

可以实现快速中英文互译。

4.9.5 vim-floaterm

它利用 Vim 的弹窗功能实现了一个内嵌终端，使用起来非常方便。

4.9.6 vim-repl

调试 Python 代码的好帮手。

4.9.7 vista.vim

是一个 LSP/ctags 的预览工具，界面清晰，功能强大。

4.9.8 vim-which-key

用于显示当前可用按键绑定。

4.9.9 ctrlsf.vim

基于 rg、ag、ack 等命令行工具实现的批量查找与替换插件，效率非常高。

4.9.10 gutentags_plus 和 vim-preview

对 Vim 的 tags 系统进行了大幅增强。

4.9.11 asyncrun 和 asynctask

是代码运行和编译的框架。

4.9.12 vim-quickui

为 Vim 添加了菜单等功能。

4.9.13 vim-help-terminal

提升了帮助文档的使用体验。

六、总结

Vim 并非“简单易用”的编辑器，但其强大的功能和效率上限使其成为技术人员的“瑞士军刀”。学习 Vim 的过程本质是掌握一种“文本操作思维”——将重复任务转化为快捷键组合，让编辑文本从“逐字修改”变为“批量操控”。尽管初期需要投入时间记忆快捷键，但长期来看，它会成为提升生产力的重要工具。

希望通过这篇博客，你能够对 Vim 有一个全面的了解，并能够在实际工作中熟练使用 Vim 进行文本编辑和代码开发。如果你有任何问题或建议，欢迎在评论区留言。
